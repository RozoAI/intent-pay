## Design Context

### Users
Developers testing the RozoAI Intent Pay SDK examples. They are evaluating payment flows, configuration behavior, and integration details quickly.

### Brand Personality
Minimal, clear, trustworthy. The interface should feel calm, direct, and easy to use rather than flashy or marketing-heavy.

### Aesthetic Direction
Light mode, restrained, utilitarian demo UI. Prioritize legibility, spacing, and obvious actions. Avoid decorative complexity and avoid making the example feel like a landing page.

### Design Principles
1. Keep the primary action obvious within a quick scan.
2. Use spacing and grouping to explain structure before using color.
3. Prefer clear panels for actionable areas and plain surfaces for supporting text.
4. Keep visual density moderate so developers can inspect flows without distraction.
5. Make configuration and payment states immediately understandable.
